# if-switch-less
Для замены конструкций switch-case лучшим выбором является Стратегия (*Strategy Pattern*).

Этот паттерн помогает снизить цикломатическую сложность кода и улучшить его читаемость,
что особенно важно при разработке **поддерживаемого программного обеспечения**.


**Цикломатическая сложность** - это показатель, который измеряет количество логических точек принятия решений в коде.

Каждый случай в switch-case увеличивает эту сложность на единицу, что негативно сказывается на поддерживаемости кода.


Принцип работы паттерна Стратегия заключается в следующем:

1. Создается интерфейс или абстрактный класс, определяющий базовый метод
2. Для каждого случая из switch-case создается отдельный класс-стратегия, реализующий этот метод
3. Вместо switch-case используется коллекция стратегий, из которой выбирается нужная в зависимости от условий

| Команда         | Описание                                                                                                                        |
|-----------------|---------------------------------------------------------------------------------------------------------------------------------|
| USER_CREATE     | Создание нового пользователя.<br/>Автоматически создаётся кошелёк для пользователя с внесённой по минимальной суммой = 500      |
| WALLET_CREATE   | Создание нового кошелька.<br/>Сумма на счету = 0                                                                                |
| WALLET_DEPOSIT  | Пополнение кошелька.<br/>Число должно быть положительное.                                                                       |
| WALLET_WITHDRAW | Вывод средств с кошелька.<br/>Число должно быть положительное.                                                                  |
| WALLET_CLOSE    | Закрытие счёта.<br/>Остаток переводится на другой счёт пользователя.<br/>Единственный кошелёк не получится удалить.             |
| WALLET_TRANSFER | Перевод средств между счетами.<br/>За переводы между своими счетами комиссия не взымается.<br/>Комиссия = 1% от суммы перевода. |
| SHOW_ALL_USERS  | Вывести весь список пользователей со всеми данными об их кошельках.                                                             |

Все команды внесены в ENUM *ConsoleOperationType*.
Каждая команда соответствует своему классу в пакете *processors*:
- реализовавшему интерфейс *OperationCommandProcessor*,
- и добавлены, в качестве бинов, в *OperationProcessorsConfiguration*.

*OperationConsoleListener* выступает в качестве основного программного интерфейса для пользователя.


Преимущества такого подхода:

- Уменьшение цикломатической сложности
- Следование принципам ООП
- Легкость добавления новых стратегий без изменения существующего кода
- Улучшение тестируемости кода


Однако стоит учитывать, что использование паттерна Стратегия требует создания дополнительных классов,
что может увеличить объем кода.

Также необходимо правильно управлять памятью при работе с объектами стратегий.
